# Lost Scrolls II

(Project/codebase formerly named "Dvergr Expanded" — renamed to reflect that this is explicitly the sequel, not a spinoff. The working directory folder has now also been renamed to match: `E:\Valheim Modding\Lost Scrolls II`. Published to GitHub at `https://github.com/yesu0725/Lost-Scrolls-II`.)

A spiritual sequel to the deprecated Thunderstore mod **Lost Scrolls** (TaegukGaming). This is not a restoration of the old mod's content — it's a continuation of its lore, rebuilt as a new vanilla-assets-only Valheim mod centered on recruitable, levelable Dvergr companions.

## Concept

Lost Scrolls told the story of Damon and his "reign of darkness," fought through stolen scrolls scattered across the world. Lost Scrolls II picks up the thread: Damon's curse never fully died — it lingers as the corruption that holds the last Dvergr in their twisted state. The scrolls in this mod are fragments of the Dvergr's own **Communion Rite**, capable of freeing a corrupted Dvergr and turning it into an ally instead of just defeating it.

Damon himself does not return as a character — see [docs/Lore.md](docs/Lore.md) for why and how his influence is handled instead.

## Core Features

- **Ally recruitment** — perform the Communion Rite on a Dvergr to free and recruit it. See [docs/Ally-Recruitment.md](docs/Ally-Recruitment.md).
- **Ally leveling** — recruited Dvergr gain XP and grow stronger over time. See [docs/Ally-Leveling.md](docs/Ally-Leveling.md).
- **Ally chores** — assign allies to vanilla workstations to automate tedious tasks. See [docs/Ally-Chores.md](docs/Ally-Chores.md).
- **Dvergr duels** — put your companion into a non-lethal "duel mode" to spar against **another player's** duel-mode companion. See [docs/Duel-Arena.md](docs/Duel-Arena.md).
- **Ally commands** — feed/heal a companion, and cycle its stance between Follow/Guard/Standby. See [docs/Ally-Commands.md](docs/Ally-Commands.md).
- **Story delivery** — lore and quest content are told through the sibling mod **Valheim ServerGuide**, not built into this mod. See [docs/ServerGuide-Integration.md](docs/ServerGuide-Integration.md).

## Hard Constraint: Vanilla Assets Only

This mod uses **only** vanilla Valheim assets and prefabs — no custom models, textures, sounds, or AssetBundles. Every recruitable Dvergr type is an existing vanilla creature with new behavior attached, not a new creature. See [docs/Technical-Constraints.md](docs/Technical-Constraints.md) before implementing anything visual or asset-related.

## Related Project

`E:\Valheim Modding\Valheim ServerGuide` is a separate, already-functional BepInEx mod that delivers narrative content via trigger-driven `guidance.yaml` chains (raven popups, messages, rune/intro text). Lost Scrolls II fires events into it rather than building its own quest-text UI. See [docs/ServerGuide-Integration.md](docs/ServerGuide-Integration.md).

## Status

All core mechanics phases (0–6) now have code in place: scaffold, lore/quest script, recruit, leveling, chores, ServerGuide integration, and non-lethal duels — plus a post-Phase-6 feature batch (hover indicators, Follow/Guard/Standby stance) and a project rename from "Dvergr Expanded" to "Lost Scrolls II" (GUID `com.lostscrollsii`, namespace `LostScrollsII`). Two feedback passes have since fixed several bugs and reworked systems: the recruited-Dvergr-still-attacks bug (root cause was the AI's `m_aggravated` flag, not faction); recruit state reverting to uncommuned after a relog (now rebuilt on spawn from the `DE_Recruited` ZDO flag via a `MonsterAI.Start` patch); feeding (now mead-based on the `G` key, detected by the potion's consume **status effect** since meads carry no `m_food`, healing a flat amount and playing the mead's own VFX); and leveling (cap raised to **10** with a rising XP curve, **biome-/HP-scaled** kill XP read live at kill time, player-kill XP, and a custom gold **`★N` badge** replacing vanilla stars — which only render 2). **None of this has been verified in a live game session** — see [docs/Testing.md](docs/Testing.md) for the full test checklist. Highest-risk unverified items: the reworked duels now need **two players** to test (cross-client `DE_Duel` replication, non-lethal, PvP-immunity, auto-stand-down); the 7 per-biome reference-HP values for XP scaling are documented, not extracted (the `[xp]` log prints live HP so they can be corrected). *(The old Phase-6 `ApplyDamage`-prefix risk is gone — non-lethal now rides on the Cecil-confirmed `Character.Damage(HitData)` prefix.)* Other open items: stance is in-memory only; Phase 6 has no arena structure (by design). **Chore assignment now persists** across relog/zone reload (chore kind + target **world position** on the companion ZDO — *not* ZDOID, which doesn't survive the connection/remap system; re-resolved by proximity on spawn) and keeps running while the owner is away — gated to the ZDO owner, with the Valheim caveat that fully-unloaded zones don't simulate (it pauses, then resumes on reload). A reported `CookingStation.IsFireLit` NRE on the **Stone Oven** was fixed by gating the fire check behind `m_requireFire` (ovens are their own heat source and leave fire-check points unconfigured).
